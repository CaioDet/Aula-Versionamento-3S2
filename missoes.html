🚀 Missões Práticas: O Seu Portfólio

Marque as caixinhas conforme for concluindo os passos!

🎨 Missão 1: A Vitrine (Front-End) ➕

Passo 1: Configurar no GitHub

  • Criar um repositório chamado portfolio-react
  • Colocar a descrição: "Meu portfólio profissional em React"
  • Marcar as 3 caixinhas: README, .gitignore (Node) e License (MIT)

Passo 2: Construir as Pastas

Crie exatamente esta estrutura usando o truque da barra ( / ):

📁 portfolio-react/ ├── 📁 src/ (Onde o código mora) │ ├── 📁 assets/ (Suas imagens) │ ├── 📁 components/ (Botões e menu) │ ├── 📁 pages/ (Telas inteiras) │ └── 📁 styles/ (Cores e CSS) ├── 📁 public/ └── 📁 docs/
  • Criei todas as pastas acima no GitHub.

Passo 3: O Conteúdo Obrigatório

  • Criar as 3 páginas principais: Home, Projetos e Contato.
  • Escrever no arquivo README.md como instalar o projeto.
  • Colocar o site no ar (Deploy via GitHub Pages ou Vercel).
⚙️ Missão 2: O Motor (Back-End) ➕

Passo 1: Configurar no GitHub

  • Criar um repositório chamado biblioteca-api
  • Colocar a descrição: "API RESTful para biblioteca em Node.js"
  • Marcar as 3 caixinhas: README, .gitignore (Node) e License (Apache 2.0)

Passo 2: A Árvore de Pastas (O truque do .gitkeep)

Atenção: O GitHub joga fora pastas vazias! Para ele salvar suas pastas, você precisa criar um arquivo chamado .gitkeep dentro de cada uma delas. Faça sua árvore ficar igual a esta:

📁 biblioteca-api/ ├── 📁 src/ │ ├── 📁 config/ │ │ └── 📄 .gitkeep │ ├── 📁 controllers/ │ │ └── 📄 .gitkeep │ ├── 📁 models/ │ │ └── 📄 .gitkeep │ └── 📁 routes/ │ └── 📄 .gitkeep ├── 📁 docs/ │ └── 📄 .gitkeep └── 📁 tests/ └── 📄 .gitkeep
  • Criei as pastas e coloquei um .gitkeep dentro de cada uma delas.
  • Criei o arquivo package.json na raiz do projeto.
🤝 Missão 3: Rotina de Profissional ➕

Nesta missão, vamos agir como uma equipe de verdade!

  • Issues: Criei 3 "tarefas" na aba Issues do GitHub (Ex: "Criar botão de login").
  • Branches: Criei um braço (Branch) separado para fazer meu código, sem mexer na branch `main`.
  • Pull Request: Abri um PR pedindo para juntar meu código rascunho com o oficial.
  • Review: Um colega da minha sala olhou meu código e aprovou.
  • Kanban: Criei um quadro na aba "Projects" dividindo as tarefas em: Fazer, Fazendo e Feito.
